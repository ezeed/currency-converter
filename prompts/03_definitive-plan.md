# Currency Converter тАФ Implementation Plan (v3 тАФ Definitive)

## Context

Build a responsive currency converter web app from scratch, matching `design-reference.png` exactly. The design shows:

- **Header**: dark background (`oklch(0.20 0.02 260)`), white `<h1>Currency Converter</h1>`, compact height
- **TitleDetailed**: purple section (`oklch(0.53 0.27 281.04)`), white text, dynamic: `"1 USD to EUR тАУ Convert US Dollar to Euro"`
- **CurrencyConverter card**: white card with `rounded-md shadow-md`, **overlaps TitleDetailed** via negative margin
  - Row: Amount input (shows `$ 1` with currency symbol prefix), From select (full name "US Dollar"), circular swap button, To select (full name "Euro")
  - Result area (two-column on desktop): left = large result + inverse rate; right = info box (light lavender bg)
  - Footer: `"US Dollar to Euro conversion тАФ Last updated Aug 8, 2025, 2:18 AM UTC"`

export const DEFAULT_AMOUNT = '1';
export const DEFAULT_FROM_CURRENCY = 'USD';
export const DEFAULT_TO_CURRENCY = 'EUR';

// Rates cache: fresh for 1 minute, then TQ refetches in background on next access.
// Balances data freshness with avoiding redundant API calls.
export const RATES_STALE_TIME_MS = 60_000; // 1 minute

// Keep cached rates in memory for 5 minutes after last subscriber unmounts.
// Swapping back to a previous pair uses cached data instantly.
export const RATES_GC_TIME_MS = 5 * 60_000; // 5 minutes

// Currency list never changes mid-session тАФ fetch once and keep forever.
export const CURRENCIES_STALE_TIME = Infinity;
export const CURRENCIES_GC_TIME = Infinity;

### Task 7 тАФ `useConversionRate` hook

`src/sections/currency-converter/hooks/useConversionRate.ts`

No debounce тАФ amount never triggers a fetch; only currency changes do. Computation is instant client-side:

```ts
export function useConversionRate(
  amount: string,
  fromCurrency: string,
  toCurrency: string
) {
  const { data: currencies } = useCurrencies();

  const { data: ratesData } = useSuspenseQuery({
    queryKey: ['rates', fromCurrency, toCurrency],
    queryFn: () => fetchRate(fromCurrency, toCurrency),
    staleTime: RATES_STALE_TIME_MS,
    gcTime: RATES_GC_TIME_MS,
  });

  const numericAmount = parseFloat(amount);
  const isValidAmount = !isNaN(numericAmount) && numericAmount > 0;
  const rate = ratesData.rates[toCurrency];

  const conversionResult: ConversionResult | undefined =
    isValidAmount && rate !== undefined
      ? {
          convertedAmount: numericAmount * rate,
          rate,
          inverseRate: 1 / rate,
          date: ratesData.date,
          fromCurrency,
          toCurrency,
          fromCurrencyInfo: currencies[fromCurrency],
          toCurrencyInfo: currencies[toCurrency],
        }
      : undefined;

  return { conversionResult, isValidAmount };
}
```

**Done when:** Amount change = instant recalc (no API call); currency change = one fetch.

### Task 10 тАФ `CurrencyConverterSection.tsx` (composition layer + state owner)

`src/sections/currency-converter/components/CurrencyConverterSection.tsx`

This component owns all state and renders both `TitleDetailed` (in the purple zone) and the white card. It lives inside the `<Suspense>` + `<ErrorBoundary>` wrapper in `App.tsx`, so it has access to resolved currency data.

```tsx
export function CurrencyConverterSection() {
  const [amount, setAmount] = useState(DEFAULT_AMOUNT);
  const [fromCurrency, setFromCurrency] = useState(DEFAULT_FROM_CURRENCY);
  const [toCurrency, setToCurrency] = useState(DEFAULT_TO_CURRENCY);

  const { data: currencies } = useCurrencies();
  const { conversionResult } = useConversionRate(
    amount,
    fromCurrency,
    toCurrency
  );

  const handleSwap = (from: string, to: string) => {
    setFromCurrency(from);
    setToCurrency(to);
  };

  useEffect(() => {
    const n = parseFloat(amount) || 1;
    document.title = `${n} ${fromCurrency} to ${toCurrency} | Currency Converter`;
  }, [amount, fromCurrency, toCurrency]);

  return (
    <>
      <TitleDetailed
        amount={amount}
        fromCurrencyInfo={currencies[fromCurrency]}
        toCurrencyInfo={currencies[toCurrency]}
      />
      <div className="-mt-15 px-4">
        <div className="mx-auto max-w-4xl">
          <div className="bg-card rounded-md p-6 shadow-md">
            <div className="flex flex-col items-end gap-4 md:flex-row">
              <AmountInput
                value={amount}
                onChange={setAmount}
                currencySymbol={currencies[fromCurrency].currency_symbol}
              />
              <CurrencySelect
                value={fromCurrency}
                onChange={setFromCurrency}
                currencies={currencies}
                label="From"
              />
              <SwapButton
                fromCurrency={fromCurrency}
                toCurrency={toCurrency}
                onSwap={handleSwap}
              />
              <CurrencySelect
                value={toCurrency}
                onChange={setToCurrency}
                currencies={currencies}
                label="to"
              />
            </div>
            <ConversionResult result={conversionResult} />
            <LastUpdated result={conversionResult} />
          </div>
        </div>
      </div>
    </>
  );
}
```

**Done when:** Title updates dynamically; swap works; card overlaps purple section visually.

## Critical Gotchas

1. **No debounce, amount not in query key** тАФ only currency changes trigger fetches; amount = instant `├Чrate` client-side
2. **`useSuspenseQuery` needs `ErrorBoundary`** тАФ always wrap; errors propagate to nearest boundary
3. **Date timezone** тАФ `new Date("2026-02-20")` = UTC midnight = wrong day in UTC-X zones; use `date + "T00:00:00"`
4. **`gcTime` not `cacheTime`** тАФ renamed in TanStack Query v5
5. **No `tailwind.config.js`** тАФ Tailwind v4 uses `@theme {}` in CSS; a config file is silently ignored
6. **`amount` state as string** тАФ `useState<number>` breaks on clear/decimal entry
7. **Coordinated overlap spacing** тАФ `TitleDetailed` `pb-*` and card wrapper `-mt-*` must be tuned together visually
8. **`useSuspenseQuery` data is never undefined** тАФ no `?? {}` guards needed in consuming components
